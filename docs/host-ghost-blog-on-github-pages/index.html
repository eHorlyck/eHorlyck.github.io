How I am hosting my Ghost blog on GitHub Pages

Feb 03, 2021 2 min read

This post is only a modification of the guide written by Bassel Cheaib here

Generating the static site

In order to generate the static site, we'll use the Ghost static site generator. With macOS, this tool can be used directly.

1. Install the tool globally

You can do that by running:

sudo npm install -g ghost-static-site-generator

2. Generate the site

  • Create an empty directory of your choice
  • Open a terminal in this directory
  • Make sure your local ghost installation is running and accessible through the browser. By default it should be running on localhost with the port 2368
  • Run the following command:
gssg --url https://username.github.io

This assumes you'll be pushing the code to your main GitHub site repository named username.github.io, where username is your username (or organization name) on GitHub.

In case you were using a custom URL or even publishing it to other providers like Netlify, change the URL accordingly. The important thing is that it matches the URL where the site will be deployed.

3. Deploy it online

By now, you should have a folder named static inside the folder you created earlier. This is where the entire static website has been generated. All you have to do is push this code to the repository and wait for it to be deployed. To make this easier and keep everithing in one repo. Go into the repo settings and set Github pages to look into docs/ instead of root/. Then we just rename the directory to docs and it works.

That's all! Whenever you create new content or update anything, all you need to do is repeat steps 3 and 4.

How I roll!

In my case the command to generate the static site and move the files in to the docs directory is:

cd ~/Development/eHorlyck.github.io && ghost start && gssg --url https://eHorlyck.github.io  && rm -rf Docs && mv static docs/

Afterwards the local Ghost instance should be stopped before adding files to git and pushing the changes.
